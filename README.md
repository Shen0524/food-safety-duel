# 🍱 食安謠言大對決

一個給食安宣導／闖關活動／課堂使用的**雙人即時對戰**網頁。
看到一句常見的食安「說法」，玩家判斷它是 **正確** 還是 **謠言**，
比誰答得又快又準。掃 QR Code 即可配對，全程匿名記錄，最後分析
**大家最常答錯的題目**。

---

## 功能

- 🎮 **建立房間**：主持方產生 4 碼房號 + QR Code
- 📱 **掃碼配對**：對手用手機掃 QR Code 直接加入，免安裝 App
- ⚡ **即時對戰**：8 題（可選 5／8／12 題），每題 20 秒，答對＋速度加分
- 🧾 **逐題解析**：每題附正解與說明，玩完即學
- 📊 **常錯題目統計**：跨所有場次累計各題答錯率，找出最容易被騙的謠言
- 📝 **對戰記錄**：每場結果存到雲端

## 設定步驟（重要）

跨裝置即時對戰需要一個雲端資料庫，這裡用 Google 的 **Firebase Realtime Database**（免費額度足夠活動使用）。

### 1. 建立 Firebase 專案
1. 到 <https://console.firebase.google.com/> 用 Google 帳號登入
2. 點「**新增專案**」，取個名字（例如 `food-safety-duel`），其餘預設即可

### 2. 啟用 Realtime Database
1. 左側選單 → **建構 Build** → **Realtime Database**
2. 點「**建立資料庫**」
3. 位置選離你近的（例如 `asia-southeast1`）
4. 安全性規則先選「**以測試模式啟動**」（方便活動使用）

> ⚠️ 測試模式的規則任何人都能讀寫，適合短期活動。活動結束後建議刪除資料庫或改規則。
> 若要稍微限制，可在「規則」貼上：
> ```json
> {
>   "rules": {
>     "rooms":  { ".read": true, ".write": true },
>     "stats":  { ".read": true, ".write": true },
>     "matches":{ ".read": true, ".write": true }
>   }
> }
> ```

### 3. 取得設定值並貼上
1. 專案首頁 → 點 **`</>`**（網頁）圖示新增一個 Web App，隨便取個暱稱
2. 它會顯示一段 `const firebaseConfig = { ... }`
3. 把整段裡的值，對應填進本資料夾的 **`firebase-config.js`**
   （**一定要有 `databaseURL` 那一行**，長得像 `https://你的專案-default-rtdb.firebaseio.com`）

### 4. 把網頁放到網路上（讓手機掃得到）
QR Code 內含的是這個網頁的網址，所以網頁必須在「網路上」、用 **https** 開啟，手機掃了才連得到。
**直接用檔案總管雙擊開啟（`file://`）只能在同一台電腦測試，手機掃不到。**

最簡單的免費方式（擇一）：

- **Firebase Hosting**（同一個專案，最順）
  1. 安裝 Node.js 後，在本資料夾執行：
     ```bash
     npm install -g firebase-tools
     firebase login
     firebase init hosting   # public 目錄選「.」(本資料夾)，single-page 選 No
     firebase deploy
     ```
  2. 部署後會給你一個 `https://你的專案.web.app` 網址，用它開啟即可

- **GitHub Pages**：把本資料夾上傳到 GitHub repo → Settings → Pages → 選 main 分支 → 取得網址

- **Netlify / Vercel**：把資料夾拖進去即可得到一個 https 網址

## 怎麼玩

1. 主持人用網址開啟頁面 → **建立對戰房間** → 出現 QR Code 與房號
2. 對手用**手機相機掃 QR Code**（或在另一台裝置點「輸入房號加入」打房號）
3. 對手加入後，主持人按 **開始對戰**
4. 雙方同時作答，看即時比分
5. 結束後看勝負與逐題解析
6. 隨時可從右上角 **📊 統計分析** 查看大家最常答錯的題目

> 想自己先測試？用兩個瀏覽器視窗（或一般視窗 + 無痕視窗）開同一個網址，
> 一邊建立房間、一邊用房號加入，就能單人測完整流程。

## 自訂題目

打開 `questions.js`，照著現有格式增刪即可：

```js
{
  id: "q19",                 // 唯一代號，不可重複
  category: "食品添加物",     // 分類（顯示與統計用）
  claim: "要判斷的說法……",    // 玩家看到的句子
  answer: false,             // true = 此說法正確 / false = 此說法是謠言
  explain: "正解說明……"      // 答完顯示
}
```

> 題庫內容為一般食安衛教整理，實際食用與健康問題仍以衛福部／食藥署等官方資訊為準。

## 常見問題

- **手機掃了 QR 沒反應？** → 網頁一定要放到網路上用 https 開啟（見步驟 4），`file://` 不行。
- **顯示「尚未設定 Firebase」？** → `firebase-config.js` 還沒填，或 `databaseURL` 漏填。
- **加入時說找不到房間？** → 房號要大小寫一致（系統會自動轉大寫），且主持人房間還沒被關閉。
- **統計沒有數字？** → 要完整玩完至少一場，統計才會累積。
